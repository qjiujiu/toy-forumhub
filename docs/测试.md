# 测试方案

本项目中包含了一些用于前端自动化测试和日志收集的机制。



## 单元测试

针对各个层级的单元测试逻辑全部写在 tests/ 目录下面



## 前后端联调

为了方便在无头浏览器 ( Headless Browser) 测试，i.e. 不使用 UI 界面的浏览器测试、自动化脚本，以及 AI 辅助开发环境中捕获前端运行时的异常，我们在前端入口注入了一个本地日志收集机制。这个机制包括两个部分：

- **前端错误拦截脚本 (`frontend/index.html`)**
  前端 `index.html` 里面的 `<head>` 标签中注入了一段脚本用于监听和捕获以下三类前端错误，一旦发生错误，该脚本会通过 POST 请求将详细信息发送到本地的 `http://127.0.0.1:9999/log` 接口。
  - `window.onerror`：全局 JavaScript 执行错误
  
  - `window.onunhandledrejection`：未捕获的 Promise 异常
  
  - `console.error`：被重写的控制台错误输出


- **本地日志服务器 (`frontend/debug/log-server.py`)**：这是一个基于 Python `http.server` 实现的轻量级后端服务，专门用于监听 9999 端口。

  - 其负责接收来自前端的错误信息，并直接打印到终端的控制台中。

  * 其内置了处理 CORS（跨域资源共享）请求的功能，确保前端请求能够成功送达。
    * `frontend/debug/puppeteer_test.js`
    * `frontend/debug/test-browser.js`
    * `frontend/debug/test.js`

此外，项目提供了一些自动化测试与调试相关的脚本 (放置于 `frontend/debug/` 目录)，用于模拟真实浏览器交互。开发者可以在终端或 CI 环境中直观地看到前端报错，而无需手动打开浏览器的开发者工具。
