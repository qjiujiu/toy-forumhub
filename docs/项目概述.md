# Toy Forumhub 项目总结

## 1. 项目简介
Toy-Forumhub 是一个基于 **FastAPI** 框架构建的轻量级论坛管理系统后端项目，实现了用户注册、发帖、评论到点赞、关注等完整的社区基础互动功能，并包含了普通用户、审核员、管理员的多角色权限划分。

**核心技术栈**

- **Web 框架**：FastAPI

- **数据库 ORM操作 **：SQLAlchemy
- **数据校验与序列化**：Pydantic

- **架构设计**：采用仓库模式 (Repository Pattern) + 依赖注入 (Dependency Injection)，不直接依赖数据的数据库

**项目架构设计与规范**

- **采用经典三层架构**
  - **业务层 (Service)**：封装核心业务逻辑，处理数据校验、调用异常等。
  - **接口层 (API)**：定义 API 路由，利用 FastAPI Depends 实现依赖注入。
  - **数据层 (Storage/Models)**：
    - **使用强类型约束的数据模型**：利用 `SQLAlchemy` 定义模型 ORM 模型，通过 BaseModel 来将原始 ORM 转为带算法的数据结构
    - **依赖抽象的接口而不依赖具体的数据库三方库**：通过接口 `IUserRepository` 规定数据库访问接口，从而不依赖具体的数据库，模式提供数据交互，便于 Mock 和单元测试。
    - **事务管理**： 通过 ContextManager 实现事务的自动提交与回滚，防止数据不一致。
    - **全局软删除**：项目强制通过 `deleted_at` 字段实现全局数据软删除机制，只允许管理员级别进行硬删除及数据恢复。

- **统一响应格式**：通过 `BizResponse` 模块，按照`data`, `msg`, `status_code` 三段式，统装所有接口的返回格式  





## 2. 核心功能模块

项目按业务域主要划分为四个核心模块：

### 2.1 用户模块 (Users & Follows)

- **用户管理**：账号注册、注销 (软删除)、多角色管理 (普通用户/审核员/管理员)
- **用户状态**：支持正常、冻结、封禁三种状态
- **关注机制**：支持用户之间的关注与取消关注
- **统计信息**：独立的用户统计表 (`user_statistics`) 用于维护粉丝数和关注数，读写分离降低主表压力。

### 2.2 帖子模块 (Posts)

- **发布与管理**：支持草稿与发布两种状态，以及多种可见性（公开、仅自己可见）。
- **动静分离设计**：
  - `posts`（静态表）：存储基础信息及发布、可见、审核等状态。
  - `post_contents`（静态表）：存储标题、长文本内容等大字段。
  - `post_stats`（动态表）：存储点赞数、评论数等高频更新字段。
- **审核机制**：审核员专属接口，实现待审、通过、拒绝状态的流转。

### 2.3 评论模块 (Comments)

- **树形/嵌套评论**：采用 `parent_id` 和 `root_id` 设计，支持一级评论与“楼中楼”回复结构。
- **先发后审策略**：用户评论直接显示，后台异步审核检测违规（支持前端折叠）。
- **动静分离**：同帖子模块，划分为 `comments` 和 `comment_contents` 表，以防止写放大。

### 2.4 点赞模块 (Likes)

- **多态点赞**：使用统一的 `likes` 表，通过 `target_type` 区分点赞对象是“帖子”还是“评论”。
- **唯一限制**：通过业务逻辑和表设计保证用户对单一目标只能点赞一次。



## 3. 待优化与未来扩展

此项目作为一个基础 Forum 模板已经拥有不错的扩展性，但可继续完善例如缓存（Redis）、鉴权 (JWT)、异步任务 (Celery) 等生产级特性。

- `docs/` 目录编写充分的需求文档，引入 docker, AI Agent, skills 自动化创建
- `frontend/` 目录维护一个风格极简风格的前端
