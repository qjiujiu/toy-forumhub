# 前端设计文档

## 1. 目录结构设计

前端代码统一放在项目根目录的 `/frontend` 中，其内部目录结构按照主流 React SPA 单页应用设计：

```shell
frontend/
├── index.html           # Vite 项目入口 HTML
├── package.json         # 项目 NPM 依赖
├── tsconfig.json        # TypeScript 编译配置
├── vite.config.ts       # Vite 构建配置
├── src/
│   ├── api/             # 自动生成的 OpenAPI API SDK（切勿手动修改）
│   ├── assets/          # 静态资源，如 SVG 图标或图片
│   ├── components/      # 复用的通用组件
│   │   ├── AppHeader.tsx# 顶部全局导航栏
│   │   └── PostCard.tsx # 复用的帖子流卡片组件
│   ├── pages/           # 页面级组件
│   │   └── Home.tsx     # 首页（最新帖子瀑布流）
│   ├── App.tsx          # 根组件，组装 Layout 骨架与 React-Router 路由表
│   └── main.tsx         # React 挂载点，同时负责引入全局 css (arco css 及 style.css)
└── static/
    └── css/
        └── style.css    # 全局自定义 CSS 覆盖与补充
```



## 2. 技术选型与工具链

本项目保持与后端 FastAPI 一样的轻量与现代性，选用了目前主流的 Web 前端技术栈来构建，并且采用了自动化的 API 接口生成方式，极大地减少了前后端对接的心智负担。

- **核心技术栈**
  - **React 18框架**：使用 Hooks 与 Function Component 进行视图层渲染和状态管理。
  - **Vite构建工具：**：采用 Vite 作为本地开发服务器和打包工具，极速的冷启动时间和热重载，替代Webpack。
  - **TypeScript 交互逻辑**：全量使用 TS 减少类型体操，保障组件与数据模型的类型安全。
  - **Arco Design (Web React) 组件库**：使用字节跳动开源的企业级组件库 [Arco Design](https://arco.design/) 打造极简、整洁的视图风格。


- 前端 API 代码自动生成

  - 后端使用的 `FastAPI` 会自动生成符合标准的 OpenAPI (Swagger) `json` 规范数据，为了彻底避免手动在前端使用 Axios 拼接 URL 或写繁琐的数据接口类型定义，在前端工具链中引入了 **`openapi-typescript-codegen`**。

  - **生成工作流：**

    - 通过 `make swagger` 命令，后端输出一份包含中文且格式化过的 `openapi.json` 及 `docs/swagger.json`。

    - 在前端目录下执行相关命令读取 `openapi.json` 并自动在 `frontend/src/api` 目录生成基于 Axios 的 SDK。随后只需要这些接口即可 拥有包含所有中英文注释 + 严格类型约束调用方法，大幅提升了开发体验。



## 3. 设计思路与规范

### 3.1 极简风格
遵循 Arco Design 默认的干净配色 (经典的蓝白与灰阶)，页面上抛弃了花哨的动画与色彩。
- **页面 Layout**：Header 吸顶设计，中端留白限定宽度居中，保证在大屏和各种设备下均有良好的可读性。
- **卡片化流式展示**：最新帖子列表使用 `<Card>` 组件渲染，帖子卡片的 `border` 与 `hover` 阴影提供了适当的 Z-index 空间感，底部带有点赞和评论的微交互。

### 3.2 访客态与用户态分离

针对论坛核心的查看和互动逻辑，目前前端的设计思路为：
- **不强制登录即可浏览**：未登录的访客可以加载并浏览所有的“公开”帖子以及帖子内部的评论。
- **互动需强制登录**：如果用户尝试在 `PostCard` 上点赞、发评论，或者点击导航栏的“注册/登录”才需要调起 Auth。

### 3.3 数据加载与体验

使用 React `useEffect` 在组件 Mount 时触发数据获取。通过提取到自定义的 Loading 态组件 `<Spin>` 避免页面在网络延时期间一片空白。并且针对没有数据的极端情况设计了优雅的 Empty 空状态。
